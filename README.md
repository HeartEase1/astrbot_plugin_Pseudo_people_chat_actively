# AstrBot 伪人主动回复插件

基于人设、对话上下文、时间规则实现的全场景智能主动回复插件。

## 功能特性

### 核心功能

1. **对话结束后智能主动回复**
   - 用户停止对话一段时间后（默认10分钟），基于对话内容和人设，智能判断是否需要主动发消息
   - 自动分析对话上下文，识别用户提到的未来场景（如"去吃饭了""去忙了"等）
   - 1小时冷却机制，避免频繁触发

2. **固定时段日常问候**
   - 每日固定时段（早中晚）发送符合人设的问候消息
   - 支持概率控制，避免过度打扰

3. **长期未活跃用户触达**
   - 1天未活跃：发送关心问候
   - 6天未活跃：发送带情绪的质问消息

4. **日常随机事件分享**
   - 基于人设生成符合角色日常的随机事件
   - 以事件为媒介发起主动聊天，增强角色真实感

### 技术特点

- ✅ 强制使用北京时间（Asia/Shanghai）
- ✅ 默认适配 QQ 平台（aiocqhttp）
- ✅ 严格的防打扰机制（默认 00:00-06:00）
- ✅ 完整的日志追踪
- ✅ 基于 SQLite 的数据持久化
- ✅ 基于 LLM 的智能决策和内容生成
- ✅ 完整的异步实现，避免阻塞
- ✅ LLM 调用超时控制（100秒）
- ✅ 完善的配置验证机制

## 安装

### 前置要求

- AstrBot >= 4.9.2
- Python >= 3.8

### 安装步骤

1. 在 AstrBot WebUI 中上传插件压缩包，或将插件文件夹复制到 AstrBot 的插件目录
2. 在 AstrBot WebUI 中启用插件
3. 插件会自动安装依赖：
   - pytz >= 2023.3
   - python-dateutil >= 2.8.2
   - apscheduler >= 3.10.0

## 工作原理

### 对话结束后主动回复流程

1. **监听消息** → 用户发送消息时启动倒计时
2. **对话结束判定** → 倒计时结束，判定对话结束
3. **上下文总结** → 调用 LLM 总结对话内容
4. **主动回复决策** → LLM 判断是否需要主动回复及最佳时间
5. **时间校验** → 避开防打扰时段，检查任务间隔
6. **任务注册** → 将任务存入数据库并注册到调度器
7. **消息生成与发送** → 到达触发时间，生成并发送消息

### 数据持久化

插件使用 SQLite 数据库存储以下数据：

- **用户基础信息**：活跃时间、触达记录、每日计数器
- **待执行任务**：任务ID、触发时间、任务类型、上下文
- **日常事件**：事件内容、触发时间、执行状态
- **消息发送记录**：用于频率限制控制

数据库文件位置：`data/plugin_data/{插件名称}/proactive_reply.db`（插件名称为metadata.yaml中的name字段）

### 定时任务

插件注册了以下全局定时任务（均为北京时间）：

- **00:00** - 重置每日计数器
- **00:30** - 生成每日问候计划
- **01:00** - 生成每日事件
- **02:00** - 清理旧消息记录（保留7天）
- **08:00** - 扫描未活跃用户

## 注意事项

1. **时区锁定**：插件强制使用北京时间，不受系统时区影响
2. **防打扰时段**：默认 00:00-06:00 不发送任何主动消息
3. **消息频率**：同一用户两条主动消息最小间隔 2 小时
4. **QQ 平台风控**：默认限制每个用户 1 小时内最多接收 2 条主动消息（每个用户独立计算）
5. **群聊场景**：默认仅在私聊生效，群聊需手动开启
6. **人设一致性**：所有主动消息严格基于用户配置的人设生成
7. **对话历史格式**：插件使用 AstrBot 的字符串格式对话历史，自动处理

## 故障排查

### 插件无法加载

- 检查 AstrBot 版本是否 >= 4.9.2
- 检查依赖是否正确安装
- 查看 AstrBot 日志中的错误信息
- 检查插件目录权限

### 主动消息未发送

- 检查插件是否启用（`enable_plugin = true`）
- 检查是否在防打扰时段（默认 00:00-06:00）
- 查看日志中的 `[LLM决策]` 是否判定需要发送
- 检查任务是否成功注册到调度器
- 检查 LLM 是否正常响应
- 检查是否触发频率限制

### 数据库错误

- 检查插件数据目录权限：`data/plugin_data/proactive_reply/`
- 如果数据库损坏，可以删除重新初始化：`data/plugin_data/proactive_reply/proactive_reply.db`
- 重启 AstrBot 让插件重新创建数据库

### LLM 调用失败

- 检查 LLM 配置是否正确
- 检查网络连接
- 查看日志中的详细错误信息
- 检查是否触发超时（默认100秒）

### 任务未执行

- 检查任务是否成功注册（查看日志）
- 检查触发时间是否正确（北京时间）
- 检查调度器是否正常运行
- 重启插件恢复待执行任务

### 技术实现

- **异步架构**：所有数据库操作使用 `asyncio.to_thread` 异步化
- **超时控制**：LLM 调用添加 100 秒超时限制
- **配置验证**：启动时验证所有配置项，自动修正无效值
- **内存管理**：插件卸载时自动清理所有对话倒计时任务，消息处理时实时清理已完成任务
- **错误处理**：完善的异常捕获和日志记录
- **时区处理**：强制使用 pytz 的 Asia/Shanghai 时区
- **数据库优化**：启用 WAL 模式提高并发性能，所有操作使用异步锁保护
- **LLM 可靠性**：添加 LLM 调用重试机制（最多2次）
- **并发控制**：使用 `asyncio.Semaphore(5)` 限制并发 LLM 调用数量
- **任务调度**：大规模任务使用分批处理，避免内存爆炸
- **线程安全**：全局异步锁保护所有数据库操作，避免数据库损坏
- **垃圾回收防护**：异步初始化任务保留强引用，避免被垃圾回收
- **分批处理**：每日事件生成和未活跃用户扫描使用分批处理，每批大小与信号量一致（5个）

### API 使用规范

本插件遵循 AstrBot 官方 API 规范：
